# Tekstura Замеры — запуск первой версии

Эта версия добавляет отдельное закрытое приложение в папку `/zamer/`.

## Что уже сделано

- Страница `/zamer/index.html`.
- Стили `/zamer/styles.css`.
- Логика Supabase `/zamer/app.js`.
- SQL-схема Supabase `supabase/zamer_schema.sql`.
- Хранилище фото `measurement-photos`.
- Архивирование через статус `Архив`, скачивание JSON/CSV.

## 1. Создать таблицы в Supabase

1. Откройте Supabase.
2. Выберите проект.
3. Откройте SQL Editor.
4. Скопируйте содержимое файла:

```text
supabase/zamer_schema.sql
```

5. Нажмите Run.

## 2. Настроить ключи Supabase

Откройте:

```text
zamer/app.js
```

Замените:

```js
const SUPABASE_URL = "PASTE_SUPABASE_URL_HERE";
const SUPABASE_ANON_KEY = "PASTE_SUPABASE_ANON_KEY_HERE";
```

На значения из Supabase:

```text
Project Settings → API → Project URL
Project Settings → API → anon public key
```

Важно: `anon public key` можно использовать на сайте. `service_role` нельзя вставлять в браузер.

## 3. Создать первого пользователя

В приложении `/zamer/` можно нажать “Создать пользователя в Supabase”.

После этого в Supabase в таблице `profiles` нужно поставить первому пользователю роль:

```sql
update public.profiles
set role = 'admin', full_name = 'Владимир'
where id = 'ID_ПОЛЬЗОВАТЕЛЯ';
```

ID пользователя можно посмотреть в Supabase Auth → Users.

## 4. Роли

```text
admin — всё
zamer — создаёт и редактирует свои замеры
constructor — проверяет замеры
manager — видит готовые замеры
viewer — только просмотр
```

## 5. Фото

Фото сохраняются в Supabase Storage bucket:

```text
measurement-photos
```

Для вашего объёма 15 замеров/месяц и 3–4 фото на замер бесплатного тарифа должно хватить на старт.

## 6. Архив

В первой версии есть:

- Скачать JSON.
- Скачать CSV.
- Перевести в архив.
- Мягко удалить.

ZIP с PDF и фото — следующий этап.
